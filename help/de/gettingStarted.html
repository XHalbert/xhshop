FIX ME!

Einen Shop eröffnen

Erstelle zunächst mit dem Pagemanager (Adminmenü ⇒ Seiten) eine neue Seite für deinen Shop. Der „normale“ Aufruf für den XH-Shop lautet:

{{{display_shop();}}}

Es ist problemlos möglich oberhalb oder unterhalb dieses Aufrufes andere Elemente auf der Shop-Seite zu platzieren.

Konfiguration/ Administration

Hopefully the screens are self-explanatory.

Adding the cart preview

You don't have to care about this. By default the cart preview is displayed on every single page and gets inserted right above the heading. Most probably you will find a better place for it. Just add 

       <?php echo $xhsCartPreview; ?>

to an appropriate place in your template.

It's not necessary, but after that you might want to remove or outcomment the lines in plugins/xhshop/index.php 

if(!strpos(file_get_contents($pth['file']['template']), '$xhsCartPreview')){
  $c[$s] = $xhsController->cartPreview() . $c[$s];
        }
to spare an unneeded call.

If you don't want to have an always visible cart preview just drop those three lines from the xhshop/index.php and link to the cart with a form like this 

<form action="?<?php echo XHS_URL; ?>" method="post">
    <input value="go to cart" type="submit">
    <input name="xhsCheckout" value="cart" type="hidden">
</form>

Adding product pages

The products got two new fields you can edit in the product editor: you can set an image and add a more detailed description.

If you add the description the xhshop will generate an internal "product page" and link to it from the cart and the product list.

If you already have created cms-pages for the products or want to present them in a more individualistic manner: just leave the description empty and xhshop will do as before: link to the first page you have selected in the form "product pages".

Adding a better shop navigation

In the default templates you will find a selectbox that offers to select the product categories. (If you have categorized your products.) That's definitely ugly - so find a nice place in your template for 

<?php echo $xhsController->shopToc(); ?>
or, if you just want to show it on the "shop"-page: 
<?php
      if($su == XHS_URL){
         echo $xhsController->shopToc();
      }
?>
… and remove the selectbox from the shop's catalog-template. (See Overwriting the frontend templates below.

This shopToc is a simple <ul>-list containing the categories. Pimp it! (The css is up to you …)

Relocate the product search input

The same for the product search input: Remove it from the catalog template and find a better place in your CMSimple-template for a form like this: 

<form action="?<?php echo XHS_URL; ?>" method="post">
    <input name="xhsProductSearch" value="" type="text" />
    <input type="submit" value="Produktsuche" />
</form>

Overwriting the frontend templates

The shop's html output is generated via the templates in the xhshop's subfolder "templates".

If you are scared to touch them - and you really should be careful - just copy the *.tpl-files you want to change for the frontend-view into the folder 

        xhshop/theme/frontend/
and play around: xhshop will have a look in there, before it opens the default template.

To get an idea what happens: Just rename the xhshop/theme/frontend/test_catalog.tpl to catalog.tpl or add some css to the theme/frontend/shopToc.tpl.
